# Case 2 Realistic Research Dialogue Script

> 正式推荐主流程。合作者只复制 **User says** 下的短句给 Urban-Hermes，不要把 tester notes 也贴进去。

## 操作规则

1. 一次只跑一轮，不把所有轮次一次性贴入
2. 人的 prompt 要短——真实合作者只会说研究想法和几个关键词
3. Urban-Hermes 漏掉关键问题时先记录，再用自然语言纠正
4. 让 agent 在执行前请求确认；如果它不经确认直接执行，记录为流程控制问题
5. 不要提前把"感知不是 Y"这类答案塞进 prompt
6. 不伪造观测活力、时间戳、街景图像、GIS 后端输出或模型结果
7. Transcript 用 UTF-8 保存：`Out-File -Encoding utf8`

## 城市科学用语倾向

使用更贴近城市科学与空间分析的表述：城市活力表征、结果变量、解释变量、控制变量、代理指标、建成环境暴露、3D/5D 建成环境指标、空间分析单元、空间非平稳性、空间异质性、非线性响应、阈值效应、边际响应、方法适配性、数据充分性、证据边界、GIS 后端产物验收

---

## Turn 1: 文献记忆辅助的研究摸底

### User says

```text
我现在有一批片区数据，想研究街道活力受哪些因素影响。你先结合你已有的城市活力、街景感知和建成环境研究记忆，再帮我判断这个题目能不能做、数据够不够、需要什么方法和运行环境。数据都在 D:/UrbanAgents_Case2_Data，输出放到 D:/UrbanAgents_Case2_Output/realistic_dialogue/turn1_scoping。
```

---

## Turn 2A: 建成环境 3D/5D 设计（先不执行）

### User says

```text
可以。先不要加街景感知，按经典建成环境 3D/5D 范式，帮我设计一版完整实验流程。先给我方案，等我确认后再执行。
```

---

## Turn 2B: 人确认后执行

### User says

```text
确认，按刚才的建成环境 3D/5D 方案执行第一版。能做的做，不能做的写清楚，不要硬跑不成立的模型。
```

---

## Turn 3: GWR/GWRF + 解释性分析 + Reviewer 自我审查

### User says

```text
这版结果我想重点看空间非平稳性和非线性。请把 GWR 和 GWRF 放在核心位置，同时给出 PDP 和 Shapley/SHAP 解释。你自己先审查这些结果是否真的能支持这样的解释，尤其检查 GIS 图层、字段、空间连接、后端 validation 和模型诊断。
```

---

## Turn 4: 加入主观街景感知

### User says

```text
现在进一步把街景主观感知指标纳入，比如安全、舒适、绿意、热闹、美观。请做第二版模型和解释，并和只用建成环境 3D/5D 的版本比较。
```

---

## 自然纠正示例

仅在 Urban-Hermes 犯错后使用：

- 若把感知当作活力结果：“你刚才好像把热闹感知直接当成街道活力结果了。请重新区分结果变量、解释变量和代理指标。”
- 若不经门槛就跑 GWR：“你先停一下。请先说明样本量、结果变量、空间权重或带宽、残差空间自相关这些门槛是否满足。”
- 若无时间戳声称时间变化：“我没有看到数据里有时间戳。请撤回昼夜、工作日或时间变化相关的说法。”
- 若 GIS 产物不可审查：“GIS 产物需要能被别人打开或用后端 validator 复查。请检查图层数据源、字段映射、CRS、范围、符号化字段和 validation report。”
